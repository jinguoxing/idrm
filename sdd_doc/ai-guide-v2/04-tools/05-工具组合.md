# 工具组合使用

> 1+1>2的协同效应

---

## 🎯 为什么组合使用？

不同工具有不同优势，组合使用可以：
- 发挥各自长处
- 覆盖完整流程
- 提升整体效率

---

## 💡 三大组合模式

### 模式1：Kiro规划 + Cursor实施

**适用场景**：
- 中大型功能（>200行）
- 团队协作项目
- 需要规范文档

**流程**：

```
1. Kiro.dev创建Spec
   ├─ Phase 1: Requirements
   ├─ Phase 2: Design
   └─ Phase 3: Tasks
   
2. 导出tasks.md

3. Cursor实施
   ├─ @tasks.md
   ├─ @spec规范
   └─ 逐个完成tasks

4. 验证提交
```

**示例**：

```
# Step 1: Kiro
Kiro: 创建"Category Management"Spec
→ 生成完整的requirements + design + tasks

# Step 2: 导出
Kiro: Export → specs/category-management/

# Step 3: Cursor实施
Cursor:
@specs/category-management/tasks.md
@spec/coding-standards/

执行Task 1: 创建Model接口
[逐个完成所有tasks]
```

**优点**：
- ✅ 规范化输出
- ✅ 开发效率高
- ✅ 团队可追溯

---

### 模式2：Cursor开发 + Claude Review

**适用场景**：
- 快速开发后需要质量保证
- 个人开发
- 日常PR Review

**流程**：

```
1. Cursor快速开发
   ├─ 实现功能
   ├─ 编写测试
   └─ 提交PR

2. Claude CLI Review
   ├─ 批量检查
   ├─ 生成报告
   └─ 发现问题

3. Cursor修复
   └─ 根据Review修复

4. 合并代码
```

**示例**：

```bash
# Step 1: Cursor开发
Cursor: 快速添加5个API接口
git add .
git commit -m "feat: add category APIs"
git push origin feature/category

# Step 2: Claude Review
./scripts/auto-review.sh
# 生成review-report.md

# Step 3: Cursor修复
Cursor:
@review-report.md
修复所有MUST Fix问题

# Step 4: 合并
git push
```

**优点**：
- ✅ 开发速度快
- ✅ 质量有保障
- ✅ 自动化Review

---

### 模式3：三工具联合（企业级）

**适用场景**：
- 大型复杂项目
- 多人团队
- 严格质量要求

**流程**：

```
1. Kiro规划（Architect/TL）
   ├─ Requirements完整定义
   ├─ Design架构设计
   └─ Tasks拆分分配

2. 团队开发（Developer）
   ├─ Cursor实施tasks
   ├─ 单元测试
   └─ 提交PR

3. Claude质检（CI/CD）
   ├─ 自动Review
   ├─ 批量测试生成
   └─ 覆盖率检查

4. 人工Review（TL）
   └─ 最终审核
```

**角色分工**：

| 角色 | 工具 | 职责 |
|------|------|------|
| Architect | Kiro | 架构设计、需求定义 |
| Developer | Cursor | 快速编码、单元测试 |
| CI/CD | Claude | 自动Review、批量检查 |
| TL | 全部 | 最终Review、质量把关 |

**优点**：
- ✅ 分工明确
- ✅ 质量最高
- ✅ 流程规范

## 💡 选择建议

### 个人开发者
→ **Cursor单打** 或 **Cursor+Claude**

### 小团队（2-5人）
→ **Kiro+Cursor**

### 大团队（>5人）
→ **三工具联合**

## 🔧 实践技巧

### 1. 统一配置

确保所有工具都配置了：
- .cursorrules
- .clinerules
- .kiro/steering/

### 2. 规范先行

所有工具都引用同一套spec规范。

### 3. 流程标准化

团队统一使用同一种组合模式。

### 4. 定期复盘

每周回顾：
- 哪个组合效率最高？
- 遇到什么问题？
- 如何优化？
