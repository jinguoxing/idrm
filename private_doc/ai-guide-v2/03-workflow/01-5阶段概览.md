# 5阶段工作流概览

> 标准化开发流程

---

## 🔄 5阶段全貌

```
Phase 0: Context (上下文)
         ↓
Phase 1: Specify (需求规范)
         ↓
Phase 2: Design (技术方案)
         ↓
Phase 3: Tasks (任务拆分)
         ↓
Phase 4: Implement (实施验证)
```

---

## 📋 各阶段详解

### Phase 0: Context

**目标**: 理解规范，准备环境

**时间**: 15-30分钟（熟手）

**活动**:
- 阅读core规范
- 理解相关架构
- 准备开发环境

**输出**: 准备就绪

---

### Phase 1: Specify

**目标**: 定义清晰需求

**时间**: 15-30分钟

**活动**:
- 编写用户故事
- EARS notation验收标准
- 定义技术约束

**输出**: `requirements.md`

**质量门禁**: Gate 1

---

### Phase 2: Design

**目标**: 创建技术方案

**时间**: 20-40分钟

**活动**:
- 架构设计
- 接口定义
- 序列图

**输出**: `design.md`

**质量门禁**: Gate 2

---

###Phase 3: Tasks

**目标**: 拆分可执行任务

**时间**: 10-20分钟

**活动**:
- 任务拆分 (<50行)
- 依赖关系
- 验收标准

**输出**: `tasks.md`

**质量门禁**: Gate 3

---

### Phase 4: Implement

**目标**: 实施和验证

**时间**: 根据任务量

**活动**:
- 编码实施
- 编写测试
- Code Review
- 集成验证

**输出**: 完整代码 + 测试

**质量门禁**: Gate 4

---

## ⚡ 简化模式

### 简单功能 (<100行)

可合并阶段：
- Phase 0 + 1: 快速Specify
- Phase 2 + 3: 简化Design
- Phase 4: 快速实施

### 复杂功能 (>500行)

完整5阶段，甚至迭代执行。

---

## 🔁 回退机制

如果发现问题：
- Phase 2 → Phase 1: 需求不清
- Phase 4 → Phase 2: 设计有缺陷

---

**标准流程，保证质量！** 📐
