# 核心理念

> Spec-Driven AI Development

---

## 💡 四大理念

### 1. Spec-Driven（规范驱动）

**理念**：先规范，后代码

**价值**：
- 统一代码风格
- 降低Review成本
- 提升代码质量

### 2. 5-Phase Workflow（标准流程）

```
Phase 0: Context → Phase 1: Specify → Phase 2: Design
         ↓                 ↓                    ↓
Phase 3: Tasks → Phase 4: Implement
```

### 3. Tool-Agnostic（工具无关）

方法论高于工具，灵活组合使用。

### 4. Quality First（质量优先）

4个质量门禁，AI代码必Review。

---

## 🎯 AI角色

✅ 助手 - 提升效率  
✅ 顾问 - 提供方案  
✅ 执行者 - 生成代码

❌ 决策者  
❌ 替代品  
❌ 保证

---

## ✅ 正确方式

DO:
1. 引用规范
2. 分阶段开发
3. Review AI代码

DON'T:
1. 直接写代码
2. 盲目信任
3. 跳过质量门禁
