## 🤔 工作流相关

### Q1: 可以跳过某些Phase吗？

**A**: 简单功能可以，但建议至少执行：
- Phase 1 (Specify)
- Phase 4 (Implement + 质量门禁)

### Q2: Phase之间如何回退？

**A**: 随时可以回退：
- 发现需求不清 → 回Phase 1
- 发现设计有问题 → 回Phase 2
- 任务拆分不合理 → 回Phase 3

### Q3: 简单功能也要走完整流程吗？

**A**: 视情况而定：
- <50行：快速Specify + Implement
- 50-200行：完整流程建议
- >200行：必须完整流程

## 🛠️ AI工具相关

### Q4: AI生成的代码可靠吗？

**A**: 需要验证：
- ✅ 通过质量门禁
- ✅ Code Review
- ✅ 测试覆盖
- ❌ 不能盲目信任

### Q5: 如何避免AI幻觉？

**A**: 
1. 引用规范 (@spec文档)
2. 分阶段开发
3. 及时Review
4. 运行测试验证

### Q6: Cursor、Claude、Kiro选哪个？

**A**: 参考决策树：
- 简单 → Cursor
- 复杂 → Kiro + Cursor
- Review → Claude CLI

## 📚 规范相关

### Q7: 规范太多记不住怎么办？

**A**: 
1. 不需要全记住
2. 用@引用让AI记住
3. 按需查阅
4. 逐步熟悉

### Q8: 规范和实际冲突怎么办？

**A**:
1. 优先遵循规范
2. 特殊情况需Tech Lead批准
3. 提议修改规范

### Q9: 如何快速查找规范？

**A**:
1. 看README索引
2. 按场景查找
3. 用AI辅助: `@spec "查找xxx规范"`

## ✅ 质量相关

### Q10: 测试覆盖率达不到80%怎么办？

**A**:
1. 优先覆盖核心业务逻辑
2. 使用AI生成测试
3. 表驱动测试提高覆盖

### Q11: Lint报错太多怎么办？

**A**:
1. 让AI修复：`@代码 "修复lint错误"` 
2. 参考coding-standards调整
3. 必要时更新.golangci.yml

### Q12: Code Review不通过怎么办？

**A**:
1. 认真阅读Review意见
2. 逐项修复
3. 学习避免重复

## 🚀 效率相关

### Q13: AI开发真的更快吗？

**A**: 
- 短期：学习成本，可能更慢
- 中期（1周后）：速度提升30-50%
- 长期：质量提升，返工减少

### Q14: 团队如何统一使用？

**A**:
1. 全员培训
2. 统一配置（.cursorrules等）
3. 定期分享最佳实践

###Q15: 如何衡量AI效果？

**A**: 跟踪指标：
- 开发时间
- 代码质量（Lint、测试覆盖率）
- Bug数量
- Review次数
